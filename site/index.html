Why Operon exists

A model can generate text.
An app needs it to do real work.

Small models are fast, private, inexpensive, and increasingly capable. But a model alone does not remember what matters, know your app state, call capabilities safely, or prove that its answer is grounded. Operon supplies that missing layer.

01 Model alone

A convincing answer—without reliable state or authority.

  • Reconstructs context from chat prose
  • Guesses when arguments are missing
  • Can produce valid-looking but unusable output
02 Model + Operon

A bounded execution—using typed context, app-owned skills, and evidence.

  • Resolves references from structured session state
  • Clarifies missing input before acting
  • Returns validated output, receipts, and a trace

A full local harness

The pieces real
AI products need.

Operon keeps language interpretation in the model and truth, permissions, state, and side effects in your application.

CTX

Typed session context

Carry the active object, view, draft, or previous result across turns without asking the model to rediscover it from prose.

MEM

Scoped durable memory

Retrieve app-approved facts and preferences by namespace, subject, sensitivity, validity, and status. Models never gain write authority.

SKL

Host-controlled skills

Expose typed capabilities with explicit schemas, confirmation requirements, and app-owned implementations.

PRE

Argument preparation

Let host code resolve “there,” “same one,” or “last result” into canonical values—or return a structured clarification.

RPL

Bounded replanning

Use one result to choose the next capability while enforcing a hard limit on dependent execution.

VAL

Validation & repair

Enforce typed output, provenance, citations, confidence, and application rules before a result reaches the user.

Context is not memory

Right information.

Operon separates what is active now from what may be remembered later and what must be fetched fresh. That makes follow-ups useful without quietly turning model output into permanent truth.

Build with Operon ↗
local terminal
from operon import Operon, SkillRegistry

runtime = Operon.wrap(
  provider=your_local_model,
  skills=SkillRegistry(app_skills),
  memory=local_memory,
)

result = runtime.run(
  "Continue with the last result",
  session_artifacts=current_context,
)
print(result.answer, result.trace.events)
